Research

I work on quantum computing — mostly the gap between the algorithms we would like to run and the noisy, resource-constrained hardware we actually have. Below are the questions I keep coming back to, and the papers they've turned into.

What I'm interested in

Quantum error correction & fault tolerance

How do we build a reliable logical qubit out of unreliable physical ones? I'm drawn to surface codes in particular — reading code distance off a lattice, minimum-weight perfect matching decoders, lattice surgery, and estimating thresholds by simulating across code distances and error rates. This is the thinking behind my SurfaceScope visualizer.

Quantum annealing & optimization

A lot of useful problems are really constrained optimization in disguise. I like figuring out how to express them as QUBOs an annealer can actually solve — encoding inequality constraints, tuning penalty weights, and contracting search regions to solve linear systems faster. Both of my journal papers came out of this line of work.

Distributed quantum computing

Near-term machines are small, so one natural question is how to spread a single logical circuit across several networked quantum nodes. At Argonne I've been partitioning and scheduling circuits across arbitrary network topologies and simulating the entanglement and remote gates that hold them together.

Benchmarking noisy hardware

Before you trust a device, you want to predict how it will behave. I'm interested in classically-simulable metrics that correlate with real hardware performance and error rates — cheap signals that tell you something true about an expensive machine.

Publications & software

PyEncode: An Open-Source Library for Structured Quantum State Preparation

Open-source library · Python · Qiskit

K. Suresh, S. Suresh

Loading a classical vector into a quantum state is the hidden bottleneck behind algorithms like HHL and QSVT — general state preparation costs O(2ᵐ) gates and erases any quantum speedup. PyEncode exploits the structure scientific vectors actually have (sinusoids, polynomials, distributions, …): you declare the pattern and it returns a verified Qiskit circuit that prepares the state in poly(m) gates, with no materialization or approximation — one function, eleven encoding patterns.
